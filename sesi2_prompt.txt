# Prompt Sesi 2: WazzapAgents Python Bridge

## Goal
Integrasikan WazzapAgents dengan WazzapSubAgents sehingga LLM2 bisa memanggil sub-agent untuk task kompleks, menerima hasilnya via webhook push (dengan fallback polling), dan menganalisis hasilnya untuk membuat reply final ke user.

## API Contract (Kontrak Komunikasi)

### Outgoing: Bridge -> SubAgent
**Method:** `POST http://localhost:5000/execute`
**Body:**
```json
{
  "session_id": "chat123_abc_1712345678",
  "instruction": "Extract tables from /storage/doc.pdf and save as CSV",
  "input_files": ["/storage/doc.pdf"],
  "callback_url": "http://localhost:8081/subagent/callback?session_id=chat123_abc_1712345678",
  "progress_webhook": "http://localhost:8081/subagent/callback?session_id=chat123_abc_1712345678&type=progress"
}
```
**Response:** `202 Accepted` -- task diterima, sedang berjalan.

### Incoming: SubAgent -> Bridge (Webhook Push)
**Endpoint yang harus dibuka:** `POST /subagent/callback` (aiohttp server port 8081)
**Payload completion:**
```json
{
  "type": "complete",
  "session_id": "chat123_abc_1712345678",
  "result": {
    "session_id": "...",
    "success": true,
    "report": "...",
    "output_files": [],
    "processing_time_sec": 42.5
  }
}
```
**Payload progress:**
```json
{
  "type": "progress",
  "session_id": "chat123_abc_1712345678",
  "entry": {
    "step": "bash:ls",
    "detail": "ls -la /storage/",
    "timestamp": 1712345678.123
  }
}
```

### Fallback Polling
**Method:** `GET http://localhost:5000/sessions/{session_id}/result`
**Digunakan hanya jika webhook tidak diterima dalam timeout 10 menit.**

  ```python
  if action_type == "execute_subtask":
      import uuid
      session_id = f"{chat_id}_{uuid.uuid4().hex[:8]}_{int(time.time())}"
      instruction = action["instruction"]
      input_files = action.get("inputFiles", [])

      task = SubTask(session_id=session_id, instruction=instruction, chat_id=chat_id)
      subagent_tracker.register(task)

      # Submit ke subagent (async, non-blocking)
      await subagent_client.submit(session_id, instruction, input_files)

      # Setup tunggu webhook
      completion_event = asyncio.Event()
      subagent_webhook.register_completion_event(session_id, completion_event)

      # TUNGGU webhook completion (max 10 menit)
      try:
          await asyncio.wait_for(completion_event.wait(), timeout=600)
      except asyncio.TimeoutError:
          # Fallback polling
          result = await subagent_client.poll_result(session_id)
          if result:
              subagent_tracker.finalize(session_id, result)
          else:
              subagent_tracker.finalize(session_id, {"success": False, "report": "Timeout waiting for sub-agent"})

      # Inject hasil ke history sebagai system message
      finalized = subagent_tracker._history.get(chat_id)
      if finalized:
          final_task = finalized[-1]
          history.append(WhatsAppMessage(
              timestamp_ms=int(time.time()*1000),
              sender="system",
              text=f"[SUBTASK FINISHED]\nResult: {final_task.report or 'No report'}\nSuccess: {final_task.status == 'completed'}",
              role="system",
          ))

      # RE-INVOKE LLM2 dengan context baru
      # Build ulang current burst atau gunakan history yang sudah diupdate
      reply_msg = await generate_reply(
          llm2_history,  # sudah termasuk hasil subtask
          current,       # bisa None atau rebuild
          current_payload=llm2_payload,
          ... # parameter lain sama
      )

      # Proses reply_msg kedua persis seperti normal (extract actions, kirim ke user)
      # PERHATIAN: Hindari infinite loop -- generate_reply kedua tidak boleh menghasilkan execute_subtask lagi.
      # Solusi: sementara override allow_subagent=False saat re-invoke, atau cek di loop.
  ```

## Acceptance Criteria
1. LLM2 tidak melihat tool `execute_subtask` kecuali jika `allow_subagent=True` (default False).
2. Saat tool dipanggil, task dikirim ke `localhost:5000/execute` dengan `callback_url` dan `progress_webhook`.
3. Webhook server di port 8081 menerima push dari SubAgent dan mengupdate tracker.
4. Progress tracking visible di context injection: setiap generate_reply selama task berjalan melihat status real-time.
5. Setelah task selesai (via webhook atau fallback polling), LLM2 di-re-invoke dengan hasil subtask di history.
6. User menerima reply final dari LLM2 yang menganalisis hasil subtask.
7. Tidak ada typing indicator (`send_typing`) yang dikirim selama subtask berjalan.
8. Jika webhook gagal (SubAgent tidak bisa reach port 8081), fallback polling tetap berfungsi dalam 10 menit.

## Catatan Penting untuk Build Agent
- **JANGAN** mengubah file `src/wa/` atau bagian Node.js dari WazzapAgents -- hanya Python Bridge.
- **JANGAN** mengubah port atau format response `GET /sessions/{id}/result` di WazzapSubAgents.
- **JANGAN** tambahkan typing indicator saat subtask berjalan.
- **PASTIKAN** `format_context()` menghasilkan output persis dengan format yang diminta user.
- **PASTIKAN** re-invoke LLM2 tidak infinite loop: either set `allow_subagent=False` saat re-invoke, atau break loop setelah re-invoke selesai.
